Lab 10

SQL injection UNION attack, retrieving multiple values in a single column

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: UNION-based SQL Injection

Objetivo

Explotar una vulnerabilidad SQL Injection en el filtro de categorías para extraer las credenciales de los usuarios desde la tabla users.

El reto consiste en que la consulta original solo devuelve una columna que acepta texto, por lo que es necesario combinar múltiples valores en un solo campo.

Vulnerabilidad

La aplicación no valida ni sanitiza correctamente el parámetro category, permitiendo inyectar consultas SQL arbitrarias.

Esto permite realizar un ataque UNION SELECT para recuperar información de otras tablas.

Procedimiento

Paso 1 – Interceptar la petición

Se filtran productos desde la página para capturar la solicitud GET con el parámetro category en Burp Suite.

Paso 2 – Confirmar número de columnas

'+UNION+SELECT+NULL,'abc'--

La respuesta muestra el texto abc, confirmando que la segunda columna acepta datos de tipo texto.

Paso 3 – Identificar la tabla de usuarios

Se identifica que la tabla vulnerable es: 

users

Paso 4 – Concatenar valores

Debido a que solo una columna acepta texto, se concatenan username y password en una sola columna: 

'+UNION+SELECT+NULL,username||'~'||password+FROM+users--

El operador || concatena los valores y el carácter ~ sirve como separador.

La página muestra resultados como: 

administrator~password123
user1~pass123

Con esto se obtienen las credenciales de administrador.

Mitigación recomendada

  • Uso de consultas parametrizadas.
  • Validación estricta de entradas.
  • Principio de mínimo privilegio en la base de datos.
  • Uso de WAF para detectar patrones de SQLi.
