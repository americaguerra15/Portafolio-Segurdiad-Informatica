Lab 18

SQL injection with filter bypass via XML encoding

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: UNION-based SQL Injection con bypass de filtros

Objetivo

Explotar una vulnerabilidad SQL Injection en la función de verificación de stock utilizando codificación XML para evadir filtros de seguridad.

El objetivo es recuperar el usuario y contraseña del administrador almacenados en la tabla users.

Vulnerabilidad

La aplicación procesa datos enviados en formato XML sin validar adecuadamente su contenido.

Además, existe un filtro que bloquea palabras clave como SELECT o UNION, pero este filtro puede ser evadido utilizando codificación XML hexadecimal.

Procedimiento

Paso 1 — Interceptar la solicitud


1
1

Paso 2 — Probar inyección básica

Se intentó el payload: 

1 UNION SELECT NULL

El servidor respondió con 403 Forbidden.

Paso 3 — Evadir filtro con XML encoding

Se utilizó la extensión Hackvertor para convertir el payload en entidades hexadecimales.

Paso 4 — Extraer credenciales

Payload utilizado: 

1 UNION SELECT username || '~' || password FROM users

La aplicación devolvió: 

administrator~PASSWORD

Con estas credenciales se inició sesión como administrador, completando el laboratorio.

Mitigación recomendada

  • Uso de consultas parametrizadas.
  • Validación estricta de entradas XML.
  • Decodificación previa antes de inspección WAF.
  • Principio de mínimo privilegio en la base de datos.
