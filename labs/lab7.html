Lab 7

SQL Injection UNION attack, determining the number of columns returned by the query

Información del laboratorio

Nivel: Apprentice

Tipo de SQLi: UNION-based SQL Injection

Objetivo

Determinar cuántas columnas devuelve la consulta SQL original utilizando un ataque UNION SELECT.

Vulnerabilidad

La aplicación concatena directamente el parámetro category dentro de la consulta SQL sin validación ni sanitización. 

SELECT * FROM products WHERE category = 'Accessories'

Procedimiento

Paso 1 – Interceptar la petición

GET /filter?category=Accessories

Se intercepta la petición usando Burp Suite.

Paso 2 – Enviar al Repeater

La solicitud se envía a Repeater para modificar el parámetro category.

Paso 3 – Probar UNION SELECT

' UNION SELECT NULL--

El servidor responde con error 500, indicando que el número de columnas no coincide.

Paso 4 – Probar con más columnas

' UNION SELECT NULL,NULL,NULL--

El servidor responde con HTTP 200 OK, confirmando que la consulta original tiene 3 columnas.

Mitigación recomendada

  • Uso de consultas parametrizadas.
  • Validación de parámetros mediante listas blancas.
  • Ocultar mensajes de error SQL.
  • Realizar auditorías de seguridad periódicas.
