Introducción

En el área de seguridad informática, el uso de firewalls es fundamental para controlar el tráfico de red y proteger los sistemas ante accesos no autorizados. En esta actividad se implementaron reglas utilizando iptables con el propósito de establecer una política de filtrado restrictiva, permitiendo únicamente conexiones necesarias para el funcionamiento de servicios esenciales como DNS, correo electrónico (SMTP) y navegación web (HTTP). De esta manera, se refuerza la importancia de aplicar configuraciones seguras que reduzcan riesgos y mantengan el control del flujo de paquetes dentro de un entorno Linux

1. Establecer una política restrictiva.

iptables -P INPUT DROP

iptables -P OUTPUT DROP

2. Permitir el tráfico de conexiones ya establecidas.

iptables -A INPUT -m state --state ESTABLISHED -j ACCEPT

3. Aceptar tráfico DNS (TCP) saliente de la red local.

iptables -A OUTPUT -p tcp --dport 53 -j ACCEPT

4. Aceptar correo entrante proveniente de Internet en el servidor de correo.

iptables -A INPUT -p tcp --dport 25 -j ACCEPT

5. Permitir correo saliente a Internet desde el servidor de correo.

iptables -A OUTPUT -p tcp --dport 25 -j ACCEPT

6. Aceptar conexiones HTTP desde Internet a nuestro servidor web.

iptables -A INPUT -p tcp --dport 80 -j ACCEPT

7. Permitir tráfico HTTP desde la red local a Internet

iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT

Conclusión

En conclusión, la configuración de reglas en iptables permite establecer un control detallado del tráfico entrante y saliente en un sistema Linux. Al definir políticas restrictivas y permitir únicamente conexiones específicas, se logra minimizar la exposición del sistema ante posibles amenazas. Además, al aceptar tráfico relacionado con conexiones establecidas, se garantiza un funcionamiento correcto sin comprometer la seguridad. Esta actividad demuestra cómo una adecuada administración de firewall contribuye directamente a la protección de servicios críticos y al fortalecimiento general de la infraestructura de red.
