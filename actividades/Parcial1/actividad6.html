Configuración de la topología de red

En esta actividad se realiza la configuración de una topología de red utilizando routers, con el objetivo de implementar un túnel VPN IPSec entre dos redes privadas. Esto permite asegurar la comunicación a través de Internet mediante cifrado, autenticación e integridad de datos.

Fases de la configuración

1. Configuración inicial

En la fase de configuración inicial se realiza la preparación básica de cada router. Primero se utiliza el comando enable (o en) para entrar al modo EXEC privilegiado. Posteriormente se utiliza configure terminal para ingresar al modo de configuración global.

Se asigna un nombre a cada router mediante el comando hostname para facilitar su identificación. Después se configuran las interfaces usando interface GigabitEthernet, asignando dirección IP y máscara con ip address. Finalmente, se activa la interfaz con no shut.

Además, se configura una ruta estática por defecto utilizando: ip route 0.0.0.0 0.0.0.0, permitiendo que el tráfico hacia redes desconocidas se envíe al siguiente salto.

2. Licencia de seguridad habilitada

En esta fase se habilita la licencia de seguridad necesaria para ejecutar funciones criptográficas como IPSec. Se activa el paquete tecnológico SecurityK9 con:

license boot technology-package securityK9

Después se guarda la configuración con copy running-config startup-config y se reinicia el equipo con reload. Finalmente se valida con show version que la licencia está activa.

3. Implementación de ACLs

En esta fase se crean Listas de Control de Acceso extendidas (ACL). Estas ACL permiten tráfico IP entre las redes privadas 192.168.1.0 y 192.168.3.0 utilizando una wildcard mask 0.0.0.255.

Estas ACL no se utilizan para bloquear tráfico, sino para definir el tráfico interesante que será protegido por IPSec. Solo los paquetes que coincidan con esta ACL serán cifrados dentro del túnel VPN.

4. Phase 1: ISAKMP policy

En la Phase 1 se configura la política ISAKMP, encargada de negociar los parámetros de seguridad antes de cifrar datos. Se utiliza el cifrado AES 256, autenticación pre-share y el grupo Diffie-Hellman 5.

Con crypto isakmp key se define la clave precompartida para autenticar al peer remoto. Esta fase establece la Security Association inicial.

5. Phase 2: IPSec transform set

En la Phase 2 se configura el transform-set de IPSec. Esta etapa define cómo se protegerán los datos dentro del túnel. Se utiliza ESP con cifrado AES 256 y autenticación con SHA-HMAC.

A diferencia de la Phase 1, aquí se cifra y autentica el tráfico real que viaja entre redes.

6. Crear el mapa criptográfico

Se crea el crypto map con el comando crypto map, el cual integra todos los elementos configurados previamente. Se define el peer remoto, el transform-set, la ACL y se habilita Perfect Forward Secrecy (PFS) con grupo 5.

También se configura el tiempo de vida de la Security Association con: set security-association lifetime seconds 86400.

7. Aplicar el mapa criptográfico

Finalmente se aplica el crypto map a la interfaz que conecta hacia la red pública. Con esto se activa IPSec en la interfaz externa y el tráfico que coincida con la ACL será cifrado automáticamente.

Conclusión

En resumen, aplicar la VPN IPSec Tunnel en la topología propuesta demuestra que la seguridad en redes necesita una configuración ordenada y coherente en cada etapa. Primero se estableció el direccionamiento IP y las rutas estáticas para asegurar la conectividad básica.

Después se habilitó la licencia SecurityK9 para activar funciones criptográficas. Las ACL definieron el tráfico interesante que sería protegido, evitando cifrar información innecesaria.

En la Phase 1, la política ISAKMP permitió negociar parámetros como AES 256, autenticación pre-share y Diffie-Hellman 5. En la Phase 2, el transform-set con ESP, AES 256 y SHA-HMAC garantizó confidencialidad e integridad.

Finalmente, el crypto map integró todo y al aplicarlo en la interfaz externa se logró establecer una conexión segura y funcional entre las dos redes privadas, asegurando autenticación, integridad y confidencialidad en la comunicación.
